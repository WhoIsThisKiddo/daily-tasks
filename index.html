#!/usr/bin/env bash
set -euo pipefail

BACKUP_DIR="$HOME/backups/$(date +%F)"
SRC_DIR="$HOME/project"
mkdir -p "$BACKUP_DIR"

# Create a timestamped tarball of the project directory
tar -czf "$BACKUP_DIR/project-$(date +%s).tar.gz" -C "$SRC_DIR" .

# Keep only the last 7 backups
ls -1t "$BACKUP_DIR"/*.tar.gz 2>/dev/null | tail -n +8 | xargs -r rm --

Ez a script változókat,set -euo pipefail beállítást, mkdir, tar, és ls|tail|xargs kombinációt használ. Feladata a projekt könyvtár tömörítése időbélyegzett tarballba és a régi mentések korlátozása (csak az utolsó 7 marad).

# List files larger than 10MB and export to CSV
$limit = 10MB
Get-ChildItem -Path "C:\Logs" -Recurse -File |
Where-Object { $_.Length -gt $limit } |
Select-Object FullName, Length, LastWriteTime |
Export-Csv -Path "$env:USERPROFILE\large-files.csv" -NoTypeInformation

A PowerShell példa pipelinet, Get-ChildItem, Where-Object és Select-Object használatát mutatja be, majd Export-Csv-vel menti az eredményt. Célja nagy fájlok felderítése és kimutatás készítése CSV formátumban.

Regex101

Feladat: Azonosíts JWT tokeneket Nginx-szerű HTTP logokból. Keressük az Authorization vagy query stringben előforduló JWT-szerű három részből álló, base64url jellegű karaktereket tartalmazó tokeneket.

Naplóminta:

192.168.1.10 - - [27/Mar/2026:09:12:45 +0000] "GET /api/user HTTP/1.1" 200 512 "-" "curl/7.68.0" "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.e30.abc123_def-456"
10.0.0.5 - - [27/Mar/2026:09:13:02 +0000] "POST /login HTTP/1.1" 302 112 "-" "Mozilla/5.0" "-"
172.16.0.2 - - [27/Mar/2026:09:14:10 +0000] "GET /data?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.ZXhhbXBsZQ.c0mpl3x-s1gn4tur3 HTTP/1.1" 200 1024 "-" "python-requests/2.25.1" "-"

Regex megoldás (keresés bárhol a log során):

(?

Magyarázat: A regex három ponttal elválasztott base64url-szerű szegmenset keres, amelyek betűket, számokat, aláhúzást és '-' karaktert tartalmazhatnak. A környezetszenzorok körül (negative lookbehind/lookahead) biztosítja, hogy a minta határolt tokenre illeszkedjen (ne legyen része hosszabb azonosítónak). Használható Nginx/Apache log parsing eszközökkel (grep -P, jq/egrep, SIEM rendszerek) a JWT-k felderítésére.
