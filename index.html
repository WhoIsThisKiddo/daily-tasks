Íme egy rövid script, amely az Nginx access logból kigyűjti azokat az IP-címeket, amelyek 5xx HTTP válaszokat kaptak, és a top 10-et listázza:

#!/usr/bin/env bash
LOG="${1:-/var/log/nginx/access.log}"
# Szűrjük a 5xx státuszokat, kinyerjük az IP-t (1. mező), megszámoljuk és rendezve kiírjuk a top 10-et
grep -E '" [5][0-9][0-9] ' "$LOG" | awk '{print $1}' | sort | uniq -c | sort -rn | head -n 10

Magyarázat: a script grep-et használ reguláris kifejezéssel a 5xx státuszok kiszűrésére, awk-val kinyeri az IP címet, sort-tal és uniq -c-vel megszámolja az ismétlődéseket, majd sort-ol és head-del korlátozza a kimenetet. A cél: gyorsan azonosítani, mely kliensek generálnak szerveroldali hibákat.

Egyszerű PowerShell parancs, amely az Application log utolsó 200 hibáját csoportosítja forrásonként és a legtöbb hibát okozó forrásokat mutatja:

$events = Get-EventLog -LogName Application -EntryType Error -Newest 200
$events | Group-Object -Property Source | Sort-Object Count -Descending | Select-Object -First 10 | Format-Table Count, Name -AutoSize

Magyarázat: a script Get-EventLog-dal lekéri az Application napló utolsó 200 hibáját, Group-Object-tal forrásonként csoportosít, Sort-Object-tal csökkenő sorrendbe rendezi a csoportokat, és Select-Object-tel kiválasztja a top 10-et. Használat: gyors áttekintés a leggyakoribb hibalefutásokról.
