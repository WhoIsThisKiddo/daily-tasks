#!/usr/bin/env bash
set -euo pipefail

# Find files larger than 100MB, archive them, and print a summary
TARGET_DIR="/var/log"
ARCHIVE="/tmp/large_logs_$(date +%Y%m%d_%H%M%S).tar.gz"

find "$TARGET_DIR" -type f -size +100M -print0 \
  | xargs -0 tar -czf "$ARCHIVE" --remove-files --warning=no-file-changed

echo "Archive created: $ARCHIVE"
echo "Contents:"
tar -tzf "$ARCHIVE"

Ez a script a következőket használja: shebang, set (ezzel szigorúbb hibakezelés), változók, find + xargs kombinációt, tar archíválást és dátum alapú fájlnevet. A cél: nagy (>100MB) fájlok megtalálása egy könyvtárban, azok becsomagolása és eltávolítása az eredeti helyükről; végül a létrejött archívum listázása.

# List top 5 processes by memory and export to CSV
Get-Process |
  Sort-Object -Property WorkingSet -Descending |
  Select-Object -First 5 -Property Id, ProcessName, @{Name='MemoryMB';Expression={[math]::Round($_.WorkingSet / 1MB,2)}} |
  Export-Csv -Path "$env:TEMP\top_memory_processes.csv" -NoTypeInformation

Write-Output "Exported to $env:TEMP\top_memory_processes.csv"

Ez a PowerShell példa alapvető parancsokat és szerkezeteket használ: Get-Process, Sort-Object, Select-Object (egy kalkulált mezővel), Export-Csv és Write-Output. A cél: a memóriát leginkább fogyasztó 5 folyamat kigyűjtése és CSV-be exportálása áttekintéshez.

Regex101

Feladat: Nginx/app logokból JWT (Bearer) tokenek automatikus felismerése és kinyerése ott, ahol Authorization fejléc vagy "Bearer" token szerepel a log sorban.

2026-03-27T09:12:33Z INFO 10.1.2.45 "GET /api/data HTTP/1.1" 200 - "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyIjoiam9lIn0.s5hXyZ-example-signature"
2026-03-27T09:13:01Z WARN 192.168.0.12 "POST /login HTTP/1.1" 401 - "Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiJ9.ABCdefGHIjklMN-opQRs.stUvwXyZ12345"
2026-03-27T09:14:02Z INFO 172.16.0.5 "GET /health HTTP/1.1" 200 - "-"
2026-03-27T09:15:44Z INFO 10.0.0.7 "GET /api/items HTTP/1.1" 200 - "Cookie: session=abc123; Authorization=Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.XYZpayload.signaturepart"

(?i)\bBearer\s+([A-Za-z0-9_-]+\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+)\b

A regex magyarázata: (?i) bekapcsolja a case-insensitivitást; \b határoló biztosítja szótagszerű egyezést; "Bearer" szó után tetszőleges szóközök; a token három ponttal elválasztott base64url-szerű részből áll (A-Za-z0-9_-)+, így reálisan felismeri a JWT formátumot log-sorokból.
