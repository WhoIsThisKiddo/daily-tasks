Egyszerű Bash script, amely végigmegy egy log fájlon és kilistázza a gyanús HTTP státuszkódokat.


#!/bin/bash

LOGFILE="access.log"

while read -r line
do
    status=$(echo "$line" | awk '{print $9}')

    if [[ "$status" == "401" || "$status" == "403" || "$status" == "500" ]]
    then
        echo "Suspicious status detected:"
        echo "$line"
        echo
    fi

done < "$LOGFILE"

A script while ciklust használ egy log fájl soronkénti feldolgozására. Az awk parancs a 9. mezőt (HTTP státuszkód) olvassa ki. Egy feltételes if blokk ellenőrzi, hogy a státuszkód 401, 403 vagy 500-e, majd kiírja a gyanús log sort.

Egyszerű PowerShell példa, amely log fájlban keres sikertelen bejelentkezési eseményeket.


$logFile = "auth.log"

Get-Content $logFile |
ForEach-Object {
    if ($_ -match "Failed password")
    {
        Write-Output "Suspicious login attempt detected:"
        Write-Output $_
        Write-Output ""
    }
}

A script a Get-Content parancs segítségével olvassa a log fájlt. A ForEach-Object minden sort feldolgoz. A -match operátor regex alapú keresést végez, és ha "Failed password" mintát talál, akkor kiírja a gyanús bejelentkezési próbálkozást.

Feladat: Találd meg az AWS Access Key ID-ket egy log fájlban. Az AWS kulcsok általában AKIA prefixszel kezdődnek és összesen 20 karakter hosszúak.


2026-03-14 09:22:11 INFO user uploaded credentials AKIAIOSFODNN7EXAMPLE
2026-03-14 09:22:15 WARN possible credential leak detected
2026-03-14 09:22:20 DEBUG token=ZXlKaGJHY2lPaUpJVXpJMU5pSjkuZXhhbXBsZQ==
2026-03-14 09:22:25 ALERT suspicious key AKIA1234567890ABCDEF used from 185.23.44.10

Regex megoldás:


AKIA[0-9A-Z]{16}

A regex az AWS Access Key ID-k mintázatát keresi. Az "AKIA" prefix után pontosan 16 darab nagybetű vagy szám szerepelhet, így a teljes kulcs hossza 20 karakter lesz.
